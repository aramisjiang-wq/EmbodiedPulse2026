## 🎯 需求背景

### 用户痛点
- **信息分散**: 具身智能领域的重要研究机构（AllenAI、NVIDIA Research、Physical Intelligence等）的论文分散在不同网站
- **追踪困难**: 需要手动访问多个网站才能了解最新研究动态
- **时效性差**: 无法及时获知这些机构的最新研究成果
- **缺乏整合**: 没有统一的平台整合这些机构的研究成果

### 目标用户
- **具身算法研究人员**: 需要追踪顶级研究机构的最新论文
- **产品经理/市场人员**: 需要了解行业前沿动态和趋势
- **投资人/分析师**: 需要跟踪行业技术发展

### 核心价值
**一句话**: 在一个平台统一追踪具身智能领域顶级研究机构的最新论文，不错过任何前沿研究。

- **机构**: Allen Institute for AI
- **研究领域**: AI、NLP、计算机视觉、具身智能
- **更新频率**: 定期发布新论文
- **数据格式**:

  - 可能提供RSS/Atom feed
  - 可能有API接口

- 论文标题
- 作者列表
- 发表年份

- **更新频率**: 高频更新
- **数据格式**:
  - 可能有API接口
  - 网页列表形式
  - 可能有筛选和搜索功能

#### 论文信息包含
- 论文标题
- 作者列表
- 发表年份
- 发表场所
- 论文链接
- 项目链接（可能）

#### 技术挑战
- 可能需要API认证
- 数据量大，需要分页处理
- 可能需要处理动态加载

**功能描述**: 用户可以查看已追踪的研究机构列表

**功能点**:
- [ ] 显示机构名称、Logo、简介
- [ ] 显示每个机构的论文总数
- [ ] 显示每个机构的最新论文更新时间
- [ ] 支持添加/删除追踪机构（管理员功能）

**数据需求**:
- 机构基本信息表
- 机构论文关联表

#### 2. 机构论文列表
**功能描述**: 按机构展示论文列表

**功能点**:
- [ ] 按机构筛选论文
- [ ] 支持多机构对比查看
- [ ] 按时间排序（最新优先）
- [ ] 支持搜索（标题、作者）
- [ ] 显示论文基本信息（标题、作者、日期、引用数）

**数据需求**:
- 论文表（扩展字段：来源机构）
- 机构表

#### 3. 机构论文详情
**功能描述**: 查看论文详细信息

**功能点**:
- [ ] 显示完整论文信息
- [ ] 显示来源机构标识
- [ ] 链接到原始论文页面
- [ ] 显示相关论文推荐

**数据需求**:
- 论文详情数据

- 抓取任务表
- 抓取日志表

**功能点**:
- [ ] 各机构论文数量趋势图

- [ ] 各机构论文引用数对比
- [ ] 时间线视图（各机构论文发布时间线）

**数据需求**:
- 论文统计数据

#### 8. 机构对比分析
- 多机构研究方向对比

## 🔍 技术可行性分析

### 数据获取方式

#### 方式1: 网页爬虫（推荐用于初期）
**优点**:
- 实现简单，快速上线
- 不依赖API，通用性强

**缺点**:
- 可能违反网站robots.txt
- 需要处理反爬虫
- 网站结构变化需要更新爬虫
- 可能不稳定

**实现方案**:
- 使用BeautifulSoup/Scrapy解析HTML
- 使用Selenium处理JavaScript渲染
- 设置合理的请求频率和User-Agent
- 实现错误重试机制

#### 方式2: API接口（如果有）
**优点**:
- 稳定可靠
- 数据格式规范
- 性能好

**缺点**:
- 可能需认证
- 可能有调用限制
- 不是所有网站都提供

**实现方案**:
- 优先查找官方API文档
- 实现API客户端
- 处理认证和限流

#### 方式3: RSS/Atom Feed（如果有）
**优点**:
- 标准格式，解析简单
- 自动更新通知

**缺点**:
- 不是所有网站都提供
- 信息可能不完整

**实现方案**:
- 使用feedparser解析
- 作为补充数据源

### 数据存储设计

#### 新增数据表

**1. research_institutes 表（研究机构表）**
```sql
CREATE TABLE research_institutes (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL UNIQUE,  -- 机构名称
    slug VARCHAR(100) NOT NULL UNIQUE,  -- URL友好标识
    website_url TEXT,  -- 官网URL
    papers_url TEXT,  -- 论文列表URL
    logo_url TEXT,  -- Logo URL
    description TEXT,  -- 机构简介
    fetch_method VARCHAR(50),  -- 抓取方式：crawler/api/rss
    fetch_config JSONB,  -- 抓取配置（API key、选择器等）
    is_active BOOLEAN DEFAULT TRUE,  -- 是否启用
    last_fetch_at TIMESTAMP,  -- 最后抓取时间
    fetch_interval_hours INTEGER DEFAULT 24,  -- 抓取间隔（小时）
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

**2. papers 表扩展字段**
```sql
ALTER TABLE papers ADD COLUMN source_institute_id INTEGER REFERENCES research_institutes(id);
ALTER TABLE papers ADD COLUMN source_url TEXT;  -- 原始论文页面URL
ALTER TABLE papers ADD COLUMN fetch_method VARCHAR(50);  -- 抓取方式
```

**3. fetch_logs 表（抓取日志表）**
```sql
CREATE TABLE fetch_logs (
    id SERIAL PRIMARY KEY,
    institute_id INTEGER REFERENCES research_institutes(id),
    fetch_type VARCHAR(50),  -- full/incremental
    status VARCHAR(50),  -- success/failed/partial
    papers_found INTEGER DEFAULT 0,  -- 发现论文数
    papers_new INTEGER DEFAULT 0,  -- 新增论文数
    papers_updated INTEGER DEFAULT 0,  -- 更新论文数
    error_message TEXT,  -- 错误信息
    started_at TIMESTAMP,
    completed_at TIMESTAMP,
    duration_seconds INTEGER
);
```

## 📊 数据需求分析

### 数据量估算

#### 初始数据
- **AllenAI**: 估计100-200篇相关论文
- **NVIDIA Research**: 估计50-100篇相关论文
- **Physical Intelligence**: 估计10-30篇相关论文
- **总计**: 约200-400篇论文（初始）

#### 增量数据
- **AllenAI**: 每月约5-10篇新论文
- **NVIDIA Research**: 每月约3-8篇新论文
- **Physical Intelligence**: 每月约1-3篇新论文
- **总计**: 每月约10-20篇新论文

### 数据质量要求

#### 必填字段
- 论文标题（必须）
- 作者列表（必须）
- 发表年份（必须）
- 来源机构（必须）
- 原始链接（必须）

#### 可选字段
- 摘要
- 发表场所
- PDF链接
- 代码链接
- 引用数（后续补充）

### 数据去重策略

#### 与现有论文去重
- **ID匹配**: 如果论文有ArXiv ID，直接匹配
- **标题相似度**: 使用现有`is_duplicate_title`函数
- **作者+标题匹配**: 作者列表和标题都相同

#### 跨机构去重
- 同一篇论文可能被多个机构发布
- 需要识别并合并

## 📝 总结

### 核心需求
**在一个平台统一追踪具身智能领域顶级研究机构的最新论文，实现自动抓取、统一展示、趋势分析。**

### 技术路线
1. **初期**: 使用网页爬虫实现基础功能
2. **中期**: 优化爬虫，支持更多机构
3. **长期**: 探索API接口，实现更稳定的数据获取

### 实现策略
1. **MVP**: 先实现1个机构的完整流程
2. **扩展**: 逐步支持更多机构
3. **优化**: 持续优化抓取稳定性和数据质量
