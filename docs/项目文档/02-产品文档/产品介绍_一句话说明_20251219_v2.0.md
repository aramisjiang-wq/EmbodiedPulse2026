## 🎯 一句话介绍

**Embodied Pulse（具身脉搏）是一个专注于具身智能和机器人领域的全栈Web应用，自动抓取、分类和展示ArXiv最新论文，追踪B站视频内容，提供研究方向活跃度可视化和活跃作者排行榜，帮助研究人员快速跟进领域进展。**

## 💻 技术架构

### 全栈技术组成

| 层次 | 技术 | 说明 |
|------|------|------|
| **前端** | HTML5 + CSS3 + JavaScript | 原生实现，无框架依赖 |
| **后端** | Flask (Python 3.9+) | Web框架 + RESTful API |
| **数据库** | PostgreSQL 15+ + SQLAlchemy | 关系型数据库 + ORM + 连接池 |
| **定时任务** | APScheduler | 自动定时抓取 |
| **数据源** | ArXiv API | 论文数据抓取 |
| **备份** | JSON文件 | 数据双重保障 |

### 数据库详情

**是的，有完整的数据库系统！**

```
数据库类型：SQLite 3
ORM框架：SQLAlchemy 2.0+
数据文件：papers.db

表结构：papers
├── id (主键) - ArXiv论文ID
├── title - 论文标题
├── authors - 作者列表
├── publish_date - 发布日期
├── update_date - 更新日期
├── pdf_url - PDF链接
├── code_url - 代码链接
├── abstract - 摘要
├── category - 研究类别（9个方向）
├── created_at - 创建时间
└── updated_at - 更新时间

索引：
• idx_category (按类别查询)
• idx_publish_date (按日期查询)
• idx_title (按标题搜索)

当前数据量：4,414 篇论文
查询性能：< 50ms (90%的请求)
```

## 💡 快速问答

**Q: 这是全栈项目吗？**
A: 是的！包含前端（HTML/CSS/JS）、后端（Flask）、数据库（SQLite）、定时任务（APScheduler）。

**Q: 数据库用的什么？**
A: SQLite 3 + SQLAlchemy ORM，当前存储4400+篇论文，支持迁移到PostgreSQL。

**Q: 为什么选SQLite？**
A: 轻量、无需单独服务器、适合中小规模数据、方便部署和备份。

**Q: 有API吗？**
A: 有！8个RESTful API端点，返回JSON格式数据。

**Q: 自动化到什么程度？**
A: 完全自动！每天凌晨2点自动抓取，智能去重，自动分类，无需人工干预。

**Q: 能处理多少数据？**
A: 当前4400+篇，设计支持10万+篇论文。

**Q: 性能如何？**
A: API响应 < 100ms，支持10+并发用户，有数据库索引优化。

**Q: 有文档吗？**
A: 有完整文档体系：PRD、Spec、使用指南、技术文档、沟通记录等20+份文档。
