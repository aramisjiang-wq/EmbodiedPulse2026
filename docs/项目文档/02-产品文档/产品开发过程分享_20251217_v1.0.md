# Embodied Pulse 产品开发过程分享

## 📖 项目概述

**Embodied Pulse**（具身脉搏）是一个面向具身智能、机器人学习、AI研究领域的论文聚合平台，旨在为研究者提供每日更新的前沿论文、新闻资讯、招聘信息和数据集资源。

### 核心价值
- **实时性**：每日自动抓取最新论文，24小时新闻更新
- **专业性**：聚焦具身智能领域，精准分类和筛选
- **互动性**：通过抽签、总结等互动功能培养用户习惯
- **一站式**：论文、新闻、招聘、数据集一站式获取

## 🎯 产品定位

### 目标用户
- 具身智能领域的研究者
- 机器人学习方向的工程师
- AI算法研究人员
- 相关领域的学生和教师

### 核心功能
1. **论文聚合**：每日自动抓取ArXiv论文，按研究方向分类
2. **新闻资讯**：24小时内的具身智能相关新闻
3. **招聘信息**：实时更新的岗位招聘信息
4. **数据集资源**：具身智能相关数据集信息
5. **互动功能**：每日抽签、研究总结等互动功能

## 🚀 开发历程

### 第一阶段：基础功能搭建
**时间**：项目初期  
**目标**：搭建核心论文抓取和展示功能

**关键决策**：
- 选择SQLite + SQLAlchemy作为数据库方案（轻量、易部署）
- 使用Flask作为后端框架（简单、灵活）
- 前端使用原生HTML/CSS/JavaScript（无框架依赖，性能好）

**技术挑战**：
- ArXiv API的限流处理
- 论文去重策略（ID去重 + 标题相似度）
- 增量更新逻辑（避免重复抓取）

**解决方案**：
- 实现延迟请求机制
- 使用`difflib.SequenceMatcher`进行标题相似度匹配
- 基于日期窗口的增量更新策略

### 第二阶段：功能扩展
**时间**：中期开发  
**目标**：扩展新闻、招聘、数据集等功能

**新增功能**：
1. **新闻模块**：
   - 多源新闻抓取（RSS、NewsAPI、Orz.ai）
   - 24小时新闻过滤
   - 自动滚动展示

2. **招聘模块**：
   - GitHub README解析
   - 自动抓取和更新
   - 时间排序优化

3. **数据集模块**：
   - Juejin文章解析
   - 结构化数据展示
   - 详细信息展示

**技术亮点**：
- 多源数据融合策略
- 时间解析和时区处理
- 数据去重和更新机制

### 第三阶段：用户体验优化
**时间**：近期开发  
**目标**：提升用户体验，培养用户习惯

**优化内容**：

   - Banner设计优化

2. **互动功能开发**：

## 💡 核心设计理念

### 1. 用户习惯培养
**问题**：如何让用户每天打开网站？  
**解决方案**：
- **早上**：抽签打卡，开启一天的研究
- **晚上**：研究总结，记录一天的收获（规划中）
- **仪式感**：金色主题、动画效果、每日限制

**效果**：通过仪式感和互动性，培养用户每日访问习惯

### 2. 主题相关性
**问题**：如何让产品更有针对性？  
**解决方案**：
- 所有功能都紧密结合具身算法领域
- 词条库使用专业术语（强化学习、机器人控制等）
- 功能设计适配实际研究场景

**效果**：用户感受到产品的专业性和针对性

### 3. 一站式服务
**问题**：如何提高用户粘性？  
**解决方案**：
- 论文、新闻、招聘、数据集一站式获取
- 无需跳转多个网站
- 信息实时更新

**效果**：用户形成使用习惯，提高回访率

## 🎨 设计决策

### 1. 为什么选择金色主题的抽签功能？
**考虑因素**：
- 金色代表温暖、仪式感
- 符合"抽签"这一传统文化元素
- 与科技感形成对比，更有趣味性

**效果**：用户反馈良好，增强了仪式感

### 2. 为什么使用渐变背景？
**考虑因素**：
- 现代感强
- 视觉层次丰富
- 符合当前设计趋势

**效果**：提升了整体视觉质量

### 3. 为什么每天只能抽一次签？
**考虑因素**：
- 增强仪式感
- 培养每日访问习惯
- 避免功能被滥用

**效果**：用户形成每日访问习惯
