## 体系概述

本论文标签体系采用**四层架构**，从感知到执行，全面覆盖具身智能领域的各个研究方向。体系结构如下：

```
论文标签体系
├── 第一层：感知层（Perception Layer）- 环境感知
├── 第二层：决策层（Decision Layer）- 决策
├── 第三层：运动层（Movement Layer）- 机器人运动
└── 第四层：操作层（Operation Layer）- 操作
```

### 体系特点

- **层次清晰**：按照从感知到执行的完整流程组织
- **覆盖全面**：涵盖具身智能领域的核心研究方向
- **标签细化**：每个层级下都有具体的子标签和技术方向
- **便于分类**：支持论文的精确分类和检索

## 第一层：感知层（Perception Layer）

**核心功能**：环境感知（Environmental Perception）

感知层负责机器人对环境的理解和感知，是具身智能的基础层。

### 1.1 2D感知（2D Perception）

专注于二维图像和视觉信息的处理。

#### 子标签

- **2D detector**：2D目标检测
  - 用于检测图像中的物体、人物等目标
  - 常见技术：YOLO、R-CNN系列、RetinaNet等

- **2D mask**：2D掩码/分割
  - 图像分割和掩码生成
  - 常见技术：Mask R-CNN、SegFormer、SAM等

- **VLM detection / caption**：视觉语言模型检测/语言标注
  - 结合视觉和语言的多模态检测
  - 图像描述生成（Image Captioning）
  - 视觉问答（Visual Question Answering）
  - 常见技术：CLIP、BLIP、LLaVA等

### 1.2 3D感知（3D Perception）

专注于三维空间信息的感知和理解。

#### 子标签

- **point cloud**：点云
  - 点云数据的处理和分析
  - 点云分类、分割、配准等
  - 常见技术：PointNet、PointNet++、Point Transformer等

- **voxel**：体素
  - 体素化表示和处理
  - 3D体素网格的构建和分析
  - 常见技术：3D CNN、VoxNet等

- **3DGS**：3D Gaussian Splatting
  - 3D高斯点渲染技术
  - 用于3D场景重建和渲染
  - 新兴的3D表示方法

- **affordance**：可操作性
  - 物体可操作性的感知
  - 识别物体可以被如何操作
  - 用于机器人抓取和操作规划

### 1.3 生成（Generation）

专注于图像和视频的生成任务。

#### 子标签

- **image/video generation**：图像/视频生成
  - 使用生成模型创建图像或视频
  - 常见技术：GAN、Diffusion Model、Stable Diffusion等
  - 应用：数据增强、场景生成等

### 1.4 理解（Understanding）

专注于场景理解和综合理解生成任务。

#### 子标签

- **scene understanding**：场景理解
  - 对复杂场景的整体理解
  - 场景解析、场景分类、场景关系理解等
  - 常见技术：场景图生成、场景解析网络等

- **understanding and generation（生成理解一体化）**：理解与生成一体化
  - 将场景理解和内容生成结合
  - 理解场景后生成相应的描述、图像或动作
  - 多模态理解与生成的联合建模

## 第二层：决策层（Decision Layer）

**核心功能**：决策（Decision Making）

决策层负责基于感知信息做出决策和规划。

### 2.1 推理（Reasoning）

专注于逻辑推理和思维链推理。

#### 子标签

- **CoT（Chain of Thought）**：思维链推理
  - 逐步推理的思维过程
  - 将复杂问题分解为多个推理步骤
  - 常见技术：Chain-of-Thought Prompting、ReAct等
  - 应用：任务规划、问题求解等

### 2.2 图建模（Graph Modeling）

专注于基于图结构的建模方法。

#### 子标签

- **semantic**：语义图
  - 语义关系的图表示
  - 知识图谱、语义网络等
  - 用于场景理解、任务规划等

### 2.3 历史（History）

专注于历史信息和记忆管理。

#### 子标签

- **memory bank**：记忆库
  - 长期记忆的存储和管理
  - 经验回放、记忆网络等
  - 用于强化学习、持续学习等场景

## 第三层：运动层（Movement Layer）

**核心功能**：机器人运动（Robot Movement）

运动层负责机器人的整体运动控制和移动。

### 3.1 全身控制（Whole-Body Control）

专注于机器人的全身协调控制。

#### 子标签

- **Humanoid**：人形机器人
  - 人形机器人的运动控制
  - 双足行走、平衡控制等
  - 常见平台：Atlas、ASIMO、Digit等

- **Loco-Manipulation**：移动操作一体化
  - 移动和操作的联合控制
  - 在移动过程中进行操作
  - 需要协调移动和操作两个子系统

- **Retarget**：运动重定向
  - 将人类或其他机器人的运动重定向到目标机器人
  - 运动映射和适配
  - 用于模仿学习和运动迁移

- **RL**：强化学习（Reinforcement Learning）
  - 使用强化学习进行运动控制
  - 策略学习、价值函数学习等
  - 常见算法：PPO、SAC、TD3等

### 3.2 移动（Locomotion）

专注于机器人的移动和导航。

#### 子标签

- **Tron**：Tron控制
  - 特定的移动控制方法
  - 可能指某种特定的移动控制算法或系统

- **quadruped**：四足机器人
  - 四足机器人的运动控制
  - 步态规划、平衡控制等
  - 常见平台：Spot、ANYmal、Laikago等

- **RL**：强化学习（Reinforcement Learning）
  - 使用强化学习进行移动控制
  - 学习步态、路径规划等
  - 常见算法：PPO、SAC等

## 第四层：操作层（Operation Layer）

**核心功能**：操作（Operation）

操作层负责机器人的具体操作任务执行。

### 4.1 遥操作（Teleoperation）

专注于远程操作和人类控制。

#### 子标签

- **VR**：虚拟现实（Virtual Reality）
  - 使用VR设备进行机器人遥操作
  - 沉浸式操作体验
  - 应用：远程操作、训练等

- **gello（外骨骼）**：外骨骼
  - 外骨骼设备的控制
  - 人体外骨骼、机器人外骨骼等
  - 用于增强人体能力或机器人控制

- **UMI**：UMI系统
  - 特定的遥操作系统或协议
  - 可能指Universal Manipulation Interface等

### 4.2 抓取（Grasp）

专注于物体抓取技术。

#### 子标签

- **Dexterous hands**：灵巧手
  - 多指灵巧手的控制
  - 精细抓取、灵巧操作等
  - 常见平台：Shadow Hand、Allegro Hand等

- **SimtoReal**：仿真到现实（Simulation to Reality）
  - 从仿真环境到现实环境的迁移
  - 域适应、sim-to-real transfer等
  - 用于降低现实世界训练成本

### 4.3 双手操作（Bimanual）

专注于双手协调操作。

#### 子标签

- **VLM planning**：视觉语言模型规划
  - 使用VLM进行双手操作规划
  - 结合视觉和语言理解进行任务规划
  - 多模态任务规划

### 4.4 VLA（Vision-Language-Action）

专注于视觉-语言-动作一体化模型。

#### 子标签

- **Lightweight**：轻量级
  - 轻量级的VLA模型
  - 模型压缩、高效推理等
  - 用于资源受限的场景

### 4.5 策略（Policy）

专注于策略学习和控制策略。

#### 子标签

- **IL**：模仿学习（Imitation Learning）
  - 从演示中学习策略
  - 行为克隆、逆强化学习等
  - 常见方法：BC、GAIL、DAgger等

- **RL**：强化学习（Reinforcement Learning）
  - 通过试错学习策略
  - 策略梯度、价值函数等
  - 常见算法：PPO、SAC、TD3等

- **Autogressive**：自回归
  - 自回归策略模型
  - 逐步生成动作序列
  - 常见技术：Transformer-based policy等

### 4.6 基准测试（Benchmark）

专注于操作任务的基准测试和评估。

#### 子标签

- **举例：Libero / maniskill等**：基准测试示例
  - **Libero**：操作任务的基准测试套件
  - **Maniskill**：操作技能评估平台
  - 其他操作相关的基准测试数据集

## 标签使用说明

### 标签层级关系

本体系采用**四层架构**，每层下有多个子标签：

```
层级1：感知层/决策层/运动层/操作层
  └── 层级2：具体研究方向（如2D、3D、Reasoning等）
      └── 层级3：具体技术标签（如2D detector、point cloud等）
```

### 标签选择原则

1. **主要标签**：根据论文的核心贡献选择最相关的层级1标签
2. **次要标签**：可以添加层级2和层级3的标签作为补充
3. **多标签**：一篇论文可以同时属于多个标签（跨层或同层不同方向）

### 标签应用场景

- **论文分类**：根据研究内容将论文归类到相应标签
- **检索筛选**：用户可以根据标签筛选感兴趣的论文
- **趋势分析**：统计各标签下的论文数量，分析研究趋势
- **推荐系统**：基于标签相似度推荐相关论文
