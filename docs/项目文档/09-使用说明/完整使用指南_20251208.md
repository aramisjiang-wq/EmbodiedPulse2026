# Robotics ArXiv Daily - 使用说明

## 🚀 快速启动

### 1. 初始化数据库（首次运行）
```bash
python3 init_database.py
```
这会：
- 创建数据库表
- 将现有 JSON 数据迁移到数据库
- 显示统计信息

### 2. 启动 Web 应用
```bash
python3 app.py
```
或使用启动脚本：
```bash
./start_web.sh
```

### 3. 访问应用
打开浏览器访问：**http://localhost:5000**

## 📊 功能说明

### 主页功能
- **统计卡片**: 显示总论文数和各类别数量
- **论文列表**: 按类别分类展示，支持标签页切换
- **论文详情**: 显示标题、作者、日期、PDF和代码链接

### 操作按钮
- **刷新数据**: 重新加载论文列表
- **抓取新论文**: 触发新的论文抓取任务

### 抓取配置
点击"抓取新论文"后可以配置：
- **每类论文数量**: 建议设置为 5-10（避免 ArXiv API 速率限制）
- **配置文件路径**: 默认为 `config.yaml`

## 🔧 数据库管理

### 查看数据库
数据库文件：`papers.db` (SQLite)

可以使用 SQLite 命令行工具查看：
```bash
sqlite3 papers.db
.tables
SELECT COUNT(*) FROM papers;
```

### 重新迁移数据
如果需要重新从 JSON 迁移到数据库：
```bash
python3 migrate_json_to_db.py
```

## ⚠️ 注意事项

1. **ArXiv API 速率限制**
   - 如果遇到 HTTP 429 错误，等待一段时间后再试
   - 建议每次抓取数量不要太大（5-10篇/类别）

2. **数据库备份**
   - JSON 文件 (`docs/cv-arxiv-daily.json`) 作为备份保留
   - 数据库文件 (`papers.db`) 建议定期备份

3. **性能优化**
   - 数据库查询比 JSON 文件快得多
   - 支持索引，查询速度更快
   - 支持并发访问

## 🐛 故障排除

### 服务器无法启动
1. 检查端口 5000 是否被占用
2. 检查依赖是否安装完整：`pip install -r requirements.txt`
3. 检查数据库是否初始化：运行 `python3 init_database.py`

### 数据加载失败
1. 检查数据库文件是否存在：`ls papers.db`
2. 如果不存在，运行初始化：`python3 init_database.py`
3. 如果数据库损坏，可以从 JSON 重新迁移

### API 返回错误
1. 查看服务器日志输出
2. 检查数据库连接
3. 如果数据库失败，会自动回退到 JSON 文件
