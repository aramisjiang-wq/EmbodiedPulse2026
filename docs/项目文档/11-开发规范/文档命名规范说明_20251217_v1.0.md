# 文档命名规范说明

## 📋 核心原则

**⚠️ 强制要求**：
- **所有文档名称必须包含日期**
- **文档更新时，必须更新文档名称中的日期**
- **这是AI的强制要求，必须时刻遵守**

## 📝 命名格式

### 标准格式
```
文档名称_YYYYMMDD.md
```

### 日期格式
- **格式**：`YYYYMMDD`（8位数字）
- **示例**：`20251212`（2025年12月12日）

### 命名示例

✅ **正确示例**：
- `Star功能方案_20251212.md`
- `数据统计方案_20251212.md`
- `AI开发原则与规范_20251212.md`
- `修复502错误_20251212.md`

❌ **错误示例**：
- `Star功能方案.md`（缺少日期）
- `数据统计方案.md`（缺少日期）
- `AI开发原则与规范_20251209.md`（日期未更新）

## 🔄 文档更新规则

### 规则1：更新时必须更新日期
- **文档内容更新时**，必须重命名文件，更新日期部分
- **示例**：
  - 旧文档：`Star功能方案_20251212.md`
  - 更新后：`Star功能方案_20251213.md`（如果今天是12月13日）

### 规则2：保留历史版本（可选）
- 如果文档有历史价值，可以保留旧版本
- 新版本使用新日期
- 旧版本保留在目录中

### 规则3：AI自动操作
- **创建新文档时**：AI自动添加当前日期
- **更新现有文档时**：AI自动更新文档名称中的日期
- **检查文档时**：AI自动检查并修复没有日期的文档

## 📂 特殊文档处理

### 可以例外的文档
以下文档可以例外（但建议也加上日期）：
- `README.md` - 索引文档（本身会定期更新）
- `CHANGELOG.md` - 发版记录（本身包含日期信息）
- `README说明.md` - 说明文档
- `图片说明.md` - 辅助说明文档

### 建议做法
即使是索引文档，也建议加上日期：
- `README_20251212.md`
- `README说明_20251212.md`

## 🔍 检查清单

### AI创建/更新文档时
- [ ] 文档名称是否包含日期？
- [ ] 日期格式是否正确（YYYYMMDD）？
- [ ] 如果是更新文档，日期是否已更新？
- [ ] 文档索引中的链接是否已更新？

### 文档更新时
- [ ] 是否重命名了文件（更新日期）？
- [ ] 是否更新了文档索引中的链接？
- [ ] 是否更新了相关文档中的引用？

## 🛠️ 批量修复工具

### 检查没有日期的文档
```bash
find docs/项目文档 -name "*.md" -type f | while read f; do
  basename "$f" | grep -qvE "[0-9]{8}" && echo "$f"
done
```

### 为文档添加日期
```bash
TODAY=$(date +"%Y%m%d")
mv "文档名称.md" "文档名称_${TODAY}.md"
```
