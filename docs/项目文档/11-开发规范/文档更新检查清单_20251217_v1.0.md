# 文档更新检查清单

## 📋 功能改动时的文档更新检查清单

**⚠️ 重要**：每次产品功能发生改动时，必须完成以下检查清单，确保所有相关文档都已同步更新。

### ✅ 必须更新的文档（功能改动时）

#### 1. PRD文档
- [ ] 文件位置：`docs/项目文档/01-需求文档/PRD_产品需求文档_YYYYMMDD.md`
- [ ] 更新功能需求描述
- [ ] 更新功能优先级
- [ ] 更新用户场景
- [ ] 更新功能规格说明
- [ ] 更新"最后更新"日期

#### 2. SPEC文档
- [ ] 文件位置：`docs/项目文档/03-技术文档/SPEC_技术规格文档_YYYYMMDD.md`
- [ ] 更新技术实现方案
- [ ] 更新API接口规范（如涉及）
- [ ] 更新数据库设计变更（如涉及）
- [ ] 更新系统架构调整（如涉及）
- [ ] 更新"最后更新"日期

#### 3. 发版记录（CHANGELOG）
- [ ] 文件位置：`docs/项目文档/05-开发记录/发版记录_CHANGELOG.md`
- [ ] 添加新的变更记录（按日期倒序）
- [ ] 更新版本号（如需要）
- [ ] 填写变更分类（✨ 功能新增、🔧 功能优化、🐛 问题修复等）
- [ ] 填写详细变更描述
- [ ] 填写变更日期

#### 4. README文档
- [ ] 文件位置：项目根目录 `README.md`
- [ ] 更新功能列表
- [ ] 更新使用说明（如涉及）
- [ ] 更新特性描述
- [ ] 更新版本号（如需要）

#### 5. 功能说明文档（如涉及）
- [ ] 文件位置：`docs/项目文档/04-功能说明/` 对应模块
- [ ] 创建或更新功能说明文档
- [ ] 包含功能概述、设计理念、技术实现、使用说明

#### 6. 文档索引
- [ ] 文件位置：`docs/项目文档/README.md`
- [ ] 更新文档列表
- [ ] 更新文档索引链接

### ✅ 其他情况下的文档更新

#### UI优化时
- [ ] 更新优化记录文档：`docs/项目文档/06-优化记录/UI优化/`
- [ ] 更新发版记录
- [ ] 更新文档索引

#### 问题修复时
- [ ] 更新问题修复文档：`docs/项目文档/08-问题修复/` 对应类型
- [ ] 更新发版记录
- [ ] 更新文档索引

#### 性能优化时
- [ ] 更新性能优化记录：`docs/项目文档/06-优化记录/性能优化/`
- [ ] 更新发版记录
- [ ] 更新SPEC文档（如涉及技术架构变更）

## 🎯 快速检查流程

### 步骤1：确认改动类型
- [ ] 功能新增
- [ ] 功能修改
- [ ] 功能删除
- [ ] UI优化
- [ ] 问题修复
- [ ] 性能优化
- [ ] 文档整理

### 步骤2：根据类型执行对应检查清单

**功能改动** → 执行"必须更新的文档"全部检查项

**UI优化** → 更新优化记录 + 发版记录 + 文档索引

**问题修复** → 更新问题修复文档 + 发版记录 + 文档索引

**性能优化** → 更新性能优化记录 + 发版记录 + SPEC文档（如涉及）

### 步骤3：验证更新完整性
- [ ] 所有相关文档都已更新
- [ ] 文档中的日期都已更新
- [ ] 文档中的版本号都已更新（如需要）
- [ ] 文档索引已更新
- [ ] 发版记录已更新

```markdown
  - 解决方案：[描述]
  - 实施细节：[描述]
  - 结果验证：[描述]
```

## ⚠️ 注意事项

1. **不要遗漏任何文档**：功能改动时必须更新PRD、SPEC、发版记录、README
2. **日期要准确**：所有文档的"最后更新"日期要一致
3. **版本号要同步**：如果更新了版本号，所有文档的版本号要一致
4. **内容要详细**：变更描述要详细，便于后续查阅
5. **链接要有效**：文档索引中的链接要确保有效

## 🔄 自动化检查

AI助手在每次功能改动后，应自动：
1. 检查PRD文档是否已更新
2. 检查SPEC文档是否已更新
3. 检查发版记录是否已更新
4. 检查README是否已更新
5. 检查文档索引是否已更新
6. 如果发现遗漏，自动补充更新
