## 📋 核心原则

### 1. 文档命名规范（强制要求）

**⚠️ 核心原则**：
- **所有文档名称必须包含日期和版本号**
- **文档更新时，必须更新文档名称中的日期和版本号**
- **这是AI的强制要求，必须时刻遵守**

#### 命名格式
- **标准格式**：`文档目的_YYYYMMDD_v版本号.md`
- **日期格式**：`YYYYMMDD`（8位数字，如：20251217）
- **版本格式**：`v主版本号.次版本号`（如：v1.0, v1.1）
- **示例**：
  - `测试记录_20251217_v1.0.md`
  - `数据问题修复记录_20251217_v1.0.md`
  - `具身论文页面功能说明_20251217_v1.0.md`
  - `部署完整指南_20251217_v1.0.md`

#### 文档更新规则
- **文档更新时，必须重命名文件，更新日期和版本号**
- **版本号规则**：
  - 主版本号：重大更新（功能重构、架构调整）
  - 次版本号：内容更新（新增章节、重要修改）
- **旧文档保留**：如果文档有历史价值，保留旧版本
- **新文档创建**：更新时创建新日期的文档，或重命名现有文档

#### AI操作要求
- **创建新文档时**：必须包含当前日期和版本号（v1.0）
- **更新现有文档时**：必须更新文档名称中的日期和版本号
- **检查文档时**：发现没有日期或版本号的文档，自动添加
- **这是AI的默认操作习惯，无需用户提醒**

### 2. 文档分类规范

#### 文件夹编号系统

所有文档按产品开发顺序编号（01-12）：

| 编号 | 文件夹名 | 阶段 | 说明 |
|------|---------|------|------|
| 01 | 需求文档 | 需求分析阶段 | 产品需求文档（PRD）、功能需求分析 |
| 02 | 产品文档 | 产品设计阶段 | 产品介绍、设计理念、开发过程分享 |
| 03 | 技术文档 | 技术设计阶段 | 技术架构、数据库设计、技术方案 |
| 04 | 功能说明 | 功能开发阶段 | **功能详细说明（按页面/功能分类）** |
| 05 | 开发记录 | 开发过程阶段 | 开发过程记录、复盘、建议 |
| 06 | 安装部署 | 安装部署阶段 | 安装指南、配置说明、环境搭建、部署方案 |
| 07 | 优化记录 | 优化迭代阶段 | UI/UX优化和性能优化记录 |
| 08 | 测试文档 | 测试验证阶段 | **测试记录和测试文档（合并）** |
| 09 | 问题修复 | 问题解决阶段 | Bug修复记录（按问题类型分类） |
| 10 | 使用说明 | 用户使用阶段 | 用户使用指南（不含部署相关） |
| 11 | 开发规范 | 规范制定阶段 | 开发原则、规范、最佳实践 |
| 12 | 沟通记录 | 沟通交流阶段 | 开发沟通历程 |

#### 文档归类原则

**1. 功能说明（04）- 按页面/功能分类**
- **原则**：功能说明文档必须归类到对应的页面或功能文件夹
- **结构**：
  ```

  │   ├── 具身论文页面功能说明_YYYYMMDD_v版本号.md
  │   └── 刷新论文数据按钮功能说明_YYYYMMDD_v版本号.md
  ├── 具身视频页面/
  │   ├── 具身视频页面功能说明_YYYYMMDD_v版本号.md
  │   └── 自然周数据维度说明_YYYYMMDD_v版本号.md
  ├── 论文抓取/
  │   └── 论文抓取流程说明_YYYYMMDD_v版本号.md
  └── 管理端/
      ├── 用户管理功能说明_YYYYMMDD_v版本号.md
      ├── 论文管理功能说明_YYYYMMDD_v版本号.md
      └── 视频管理功能说明_YYYYMMDD_v版本号.md
  ```

**2. 测试文档（08）- 测试记录和测试文档合并**
- **原则**：测试记录和测试文档是一回事，统一归类到08-测试文档
- **命名**：`测试记录_YYYYMMDD_v版本号.md`
- **内容**：包含所有测试报告、测试指南、测试结果

**3. 问题修复（09）- 按问题类型分类**
- **结构**：

**4. 优化记录（07）- 按优化类型分类**
- **结构**：
  ```

  ```

### 3. 文档合并规范

#### 合并原则

**相同目的的文档应合并**，避免文档分散：

1. **测试文档合并**：
   - 合并为：`测试记录_YYYYMMDD_v版本号.md`
   - 包含：所有测试报告、测试指南、测试结果

2. **问题修复文档合并（按问题类型）**：
   - 数据问题：`数据问题修复记录_YYYYMMDD_v版本号.md`
   - 功能问题：`功能问题修复记录_YYYYMMDD_v版本号.md`
   - UI问题：`UI问题修复记录_YYYYMMDD_v版本号.md`

3. **开发记录合并**：
   - 合并为：`开发记录_YYYYMMDD_v版本号.md`
   - 包含：所有开发过程记录、复盘、总结

#### 合并时机

- 当同一目的的文档超过3个时，考虑合并
- 当文档内容重复度超过50%时，必须合并
- 合并时保留重要信息，删除重复内容
- 合并后更新文档名称中的日期和版本号

### 4. 文档更新规范

#### 功能改动时的文档更新清单（必须执行）

**当涉及产品功能改动时（新增、修改、删除功能），必须同步更新：**

1. **PRD文档**（必须）
   - 文件位置：`docs/项目文档/01-需求文档/PRD_产品需求文档_YYYYMMDD_v版本号.md`
   - 更新内容：功能需求描述、功能优先级、用户场景、功能规格说明

2. **SPEC文档**（必须）
   - 文件位置：`docs/项目文档/03-技术文档/SPEC_技术规格文档_YYYYMMDD_v版本号.md`
   - 更新内容：技术实现方案、API接口规范、数据库设计变更

3. **CHANGELOG文档**（必须）
   - 文件位置：`docs/项目文档/05-开发记录/发版记录_CHANGELOG.md`
   - 更新内容：版本号、更新日期、功能变更说明

4. **README文档**（必须）
   - 文件位置：`README.md`（根目录）
   - 更新内容：项目介绍、功能列表、快速开始

5. **相关功能说明文档**（必须）
   - 文件位置：`docs/项目文档/04-功能说明/具体页面或功能/`
   - 更新内容：功能详细说明、使用指南

### 5. 文档索引规范

#### README.md维护

- **位置**：`docs/项目文档/README.md`
- **作用**：文档索引，快速查找文档
- **更新要求**：
  - 新增文档时，必须更新README.md
  - 文档重命名时，必须更新README.md中的链接
  - 文档删除时，必须从README.md中移除

#### 文档链接规范

- 使用相对路径：`./04-功能说明/具身论文页面/具身论文页面功能说明_20251217_v1.0.md`
- 链接必须指向最新版本的文档
- 文档重命名后，必须更新所有引用链接
