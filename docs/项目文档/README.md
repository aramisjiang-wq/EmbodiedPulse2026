## 📝 文档维护原则

### 文档分类
- **核心文档** ⭐ - 重要且经常使用的文档
- **参考文档** - 详细的技术参考
- **历史文档** - 归档到"归档文档"目录

### 文档命名规范
- 格式：`功能名称_版本号.md`
- 版本号：`YYYYMMDD_vX.X`
- 示例：`B站数据问题完整解决方案_20251219_v2.0.md`

### 文档更新原则
1. **一个目的一个文档** - 相同目的的文档合并
2. **删除冗余** - 删除重复和过时的文档
3. **归档历史** - 历史文档移到归档目录
4. **保持更新** - 及时更新文档内容
