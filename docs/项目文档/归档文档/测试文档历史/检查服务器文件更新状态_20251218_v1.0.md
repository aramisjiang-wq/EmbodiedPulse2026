# 检查服务器文件更新状态

**日期**: 2025-12-18

---

## 问题

字段命名已经匹配，但前端仍然不显示数据。可能是服务器上的文件没有更新。

---

## 检查步骤

### 1. 检查服务器上的 bilibili.html 文件

在服务器上运行：

```bash
cd /srv/EmbodiedPulse2026

# 检查文件是否包含"总视频"（这是新添加的）
grep -n "总视频" templates/bilibili.html

# 如果找不到，说明文件需要更新
# 应该在第1826行附近看到"总视频"
```

### 2. 检查 bilibili_models.py 文件

```bash
# 检查是否包含 views_formatted 和 views_count 字段
grep -n "views_formatted\|views_count" bilibili_models.py

# 应该在第62-64行看到这些字段
```

---

## 如果文件需要更新

### 方案1：使用 scp 同步文件（如果有SSH访问）

```bash
# 在本地执行
scp templates/bilibili.html user@server:/srv/EmbodiedPulse2026/templates/
scp bilibili_models.py user@server:/srv/EmbodiedPulse2026/
```

### 方案2：直接在服务器上修改文件

如果无法同步，可以在服务器上直接修改文件。

---

## 验证修复

更新文件后：
1. 重启应用
2. 清除浏览器缓存
3. 硬刷新页面（Ctrl+Shift+R）
4. 检查是否显示总播放数和总视频数
