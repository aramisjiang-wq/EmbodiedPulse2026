# 清除缓存和验证修复

## 问题说明

数据修复已完成，但前端可能因为以下原因没有立即显示：
1. **API缓存**：后端有5分钟缓存
2. **浏览器缓存**：浏览器可能缓存了旧数据
3. **服务未重启**：Flask服务可能还在使用旧代码

## 解决方案

### 方法1：清除API缓存（推荐）

在浏览器中访问以下URL，添加`?force=1`参数强制刷新：

```
https://blibli.gradmotion.com/bilibili?force=1
```

或者在开发者工具中，访问API接口：

```
https://blibli.gradmotion.com/api/bilibili/all?force=1
```

### 方法2：重启Flask服务

```bash
sudo systemctl restart embodiedpulse
```

### 方法3：清除浏览器缓存

1. 按 `Ctrl+Shift+R` (Windows/Linux) 或 `Cmd+Shift+R` (Mac) 强制刷新
2. 或者在开发者工具中，右键刷新按钮，选择"清空缓存并硬性重新加载"

## 验证修复

修复后应该看到：

1. **视频数量**：显示正确的数字（49、50、35等），而不是大数字（3307108等）
2. **总播放量**：保持不变（330.7万、1844.6万等）

## 如果仍然显示不正确

1. 检查浏览器控制台是否有错误
2. 检查API返回的数据：
   ```javascript
   // 在浏览器控制台执行
   fetch('/api/bilibili/all?force=1')
     .then(r => r.json())
     .then(data => console.log(data.data[0].user_stat))
   ```
3. 确认返回的`user_stat.videos`是正确的视频数量
