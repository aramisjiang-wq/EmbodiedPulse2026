# 文档管理规范

## 📋 规范说明

本文档规定了项目文档的命名、分类和管理规范，确保文档结构清晰、易于查找和维护。

## 📁 文档分类

所有文档按以下类型分类：

### 📖 使用说明 (`使用说明/`)
- 用户使用指南
- 操作手册
- 快速入门教程
- Web应用使用说明

### 🔧 功能说明 (`功能说明/`)
- 功能特性详细说明
- API接口文档
- 配置说明
- 功能使用指南

### 💻 技术文档 (`技术文档/`)
- 技术架构文档
- 数据库设计
- 系统设计文档
- 技术总结

### 🧪 测试文档 (`测试文档/`)
- 测试结果报告
- 测试用例文档
- 测试计划

### 🐛 问题修复 (`问题修复/`)
- Bug修复记录
- 问题解决方案
- 故障排查文档

### ✨ 优化记录 (`优化记录/`)
- UI/UX优化记录
- 性能优化记录
- 功能优化总结

## 📝 文档命名规范

### 命名格式
```
文档名称_YYYYMMDD.md
```

### 命名规则
1. **文档名称**：使用中文，简洁明了，能准确描述文档内容
2. **日期格式**：YYYYMMDD（年月日，8位数字）
3. **分隔符**：使用下划线 `_` 分隔文档名称和日期
4. **文件扩展名**：`.md`（Markdown格式）

### 命名示例
- ✅ `使用说明_20251208.md`
- ✅ `论文抓取逻辑说明_20251208.md`
- ✅ `UI优化说明_20251208.md`
- ❌ `使用说明.md`（缺少日期）
- ❌ `使用说明-2025-12-08.md`（日期格式错误）

## 🔄 文档更新流程

### 创建新文档
1. **确定文档类型**：根据文档内容确定应放入哪个分类目录
2. **命名文档**：使用标准命名格式 `文档名称_YYYYMMDD.md`
3. **放置位置**：放入对应的分类目录
4. **更新索引**：
   - 更新 `docs/项目文档/README.md`
   - 更新主 `README.md` 的文档索引部分

### 更新现有文档
1. **保留历史**：原文档保留作为历史记录
2. **创建新版本**：使用当前日期创建新版本 `文档名称_YYYYMMDD.md`
3. **更新索引**：更新文档索引，指向最新版本

### 文档归档
- 超过3个月的文档可以归档到 `docs/项目文档/归档/` 目录
- 归档时保持原有目录结构

## 📅 日期获取

### 命令行获取日期
```bash
date +"%Y%m%d"  # 输出：20251208
```

### Python获取日期
```python
from datetime import datetime
date_str = datetime.now().strftime("%Y%m%d")
```

## 🔍 文档查找

### 按类型查找
- 查看 `docs/项目文档/README.md` 了解各类型文档列表

### 按日期查找
- 使用文件管理器按日期排序

- 使用 `grep -r "关键词" docs/项目文档/`

## 📝 示例

### 创建新文档示例

假设要创建一个新的"部署说明"文档：

1. **确定类型**：属于"使用说明"
2. **命名**：`部署说明_20251208.md`
3. **放置**：`docs/项目文档/使用说明/部署说明_20251208.md`
4. **更新索引**：
   - 在 `docs/项目文档/README.md` 的"使用说明"部分添加
   - 在主 `README.md` 的"使用说明"部分添加

### 更新文档示例

假设要更新"使用说明"文档：

1. **保留原文档**：`使用说明_20251208.md`（保持不变）
2. **创建新版本**：`使用说明_20251215.md`（使用新日期）
3. **更新索引**：指向最新版本

## ⚠️ 注意事项

1. **不要删除旧文档**：保留历史记录，便于追溯
2. **统一命名格式**：确保所有文档遵循相同规范
3. **及时更新索引**：每次创建或更新文档都要更新索引
4. **文档分类准确**：确保文档放入正确的分类目录
